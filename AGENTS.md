# AGENTS

## Общие принципы
- Проект работает на 1С-Битрикс; пользовательский код живет в каталоге `local`, системный `bitrix` не правим без крайней необходимости.
- Сохраняйте стиль PHP из существующих файлов: короткие теги `<? ... ?>`/`<?= ... ?>`, защита `B_PROLOG_INCLUDED` в компонентах и шаблонах.
- Перед обращением к API подключайте модули через `Loader::includeModule` или `CModule::IncludeModule`, валидируйте входные параметры и делайте ранний `return` при ошибках.
- В собственных компонентах используйте кеширование `StartResultCache` и настраивайте `CACHE_TIME` аналогично текущей реализации.
- Подключайте стили и скрипты через `Bitrix\Main\Page\Asset`, не дублируя `<link>`/`<script>` в шаблонах вручную.
- Для общих вспомогательных функций опирайтесь на `local/php_interface/init.php` (`getSettings`, `num_word_standart`, `splitByParentheses`) вместо дублирования.
- Применяйте `Loc::loadMessages` и языковые файлы для новых текстов в шаблонах и компонентах.

## Структура верстки и компонентов
- Основной шаблон лежит в `local/templates/fors`; сохраняйте существующие include-блоки (`includes/header-*.php`, `template_blocks/*`) и классы верстки.
- Хлебные крошки выводятся компонентом `bitrix:breadcrumb`; при добавлении страниц следите за совместимостью с текущими обертками `<main>`/`section`.
